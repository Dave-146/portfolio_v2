UX/UI Case Study

Harmony: Redesigning the Music Experience

A comprehensive redesign of a music streaming platform focused on enhancing user engagement and accessibility while maintaining brand identity.

Timeline 12 Weeks
My Role Lead Designer
Tools Figma, Miro
Scroll to explore

Project Overview

Harmony, a growing music streaming service, faced challenges with user retention and engagement. My task was to redesign their mobile application to improve the user experience while maintaining their brand identity.

The goal was to create an intuitive, visually appealing interface that would encourage longer session times and increase user satisfaction.

Increased user engagement by 47% after implementation

Reduced user drop-off rate by 32% during onboarding

Improved accessibility score from 76% to 94%

The Challenge & Solution

The Challenge

Users were finding it difficult to discover new music and navigate through their existing libraries. The previous interface was cluttered and lacked visual hierarchy.

User research revealed that 68% of users were frustrated with the recommendation system, and 54% found the navigation confusing.

The Solution

I redesigned the interface with a focus on content discovery and personalization. The new design features an intuitive navigation system and a visually appealing layout.

The recommendation algorithm was enhanced with visual cues to help users understand why certain content was being suggested to them.

Key Features & Improvements

Personalized Recommendations

Enhanced algorithm with visual explanations for why content is recommended, helping users discover new music aligned with their tastes.

Streamlined Player Interface

Redesigned music player with intuitive controls, visual feedback, and seamless transitions between browsing and listening.

Smart Library Organization

Intelligent organization system that automatically categorizes music and creates smart playlists based on listening habits.

Accessibility Improvements

Enhanced Color Contrast

Improved contrast ratios throughout the interface to meet WCAG AA standards.

Dynamic Text Sizing

Support for system text size preferences and scalable interface elements.

Voice Control Integration

Comprehensive voice command support for hands-free operation.

Keyboard Navigation

Improved focus states and keyboard shortcuts for non-touch navigation.

Lessons Learned

Key Takeaways

This project reinforced several important principles and provided valuable insights for future work:

User-centered design is paramount. The most successful features were those directly addressing user pain points identified during research.

Iteration is essential. Multiple rounds of testing and refinement were crucial to achieving the final design quality.

Accessibility benefits everyone. Features designed for accessibility improved the experience for all users, not just those with specific needs.

Visual design and functionality must work together. The aesthetic improvements enhanced usability rather than competing with it.

What I Would Do Differently

If I were to approach this project again, I would:

Involve developers earlier in the process to better understand technical constraints.

Conduct more international user testing to ensure the design works across different cultural contexts.

Create a more comprehensive design system from the start to improve consistency and development efficiency.
